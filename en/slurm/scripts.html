Job scripts#

Français

A job script is commonly a Bash script intended for the Slurm scheduler. It includes:

  1. The shebang on the very first line. For example: #!/bin/bash.

  2. The #SBATCH options for the job. These options will be read by the sbatch job submission command.

  3. Loading required modules.

  4. Bash commands that will be executed automatically on CPU cores reserved for the job.

See examples from the Alliance documentation:

  • Simple script

  • Python job

  • R job

Exercise - Simple script#

  1. Log in to the virtual cluster with ssh login1.

  2. Go to the exercise directory with cd ~/cq-formation-cip201-main/lab/script.

  3. In the simple.sh file, edit the #SBATCH options according to the instructions there:

    1. Request an amount of memory.

    2. Set a time limit for execution.

    Note: The scheduler will automatically select the def-sponsor00 compute account.

  4. Submit this job script using the sbatch command.

  5. Monitor the job status with the sq command.

  6. Once the job is completed, see the contents of the output file (slurm-<jobid>.out) with less (press q to exit).
